Procedimentos Internos de Suporte a API

1. Verificação do Servidor CERMOB.

Acesse o link do servidor CERMOB para verificar se está ativo:

  • https://certeduc.com:45143/system

2. Realizar login na API via Postman.

Utilize o Postman para realizar o login na API utilizando o CNPJ/CPF conforme documentação fornecida.

  • https://www.postman.com//

  • Tela do Postman teste de API Login e senha ("user", CNPJ ou CPF) "pw".

3. Procedimentos válidos para RD e GED.

Os procedimentos deste teste são válidos para os módulos GED (Acervo Digital) e RD (Registro Diplomas).

4. Acessar o Portal e tentar executar alguma ação.

Acesse o portal correspondente à API e execute alguma ação que envolva interação com os serviços providos pela API.

5. Coleta de informações e Testes Realizados.

Após realizar todas as ações acima, reúna todas as informações relevantes e detalhes dos testes realizados para compor o ticket.

  • Dados de login (CNPJ ou CPF) e senha do cliente, os dados podem ser encontrados na rede interna, dentro da pasta projetos clientes .
  • Numero do lote para problemas no envio de lotes para o portal.
  • Arquivos Json ou XML.
  • Qual o ambiente Produção / Homologação.
  • Prints de tela exibindo a mensagem de erro se possível com data e hora.

    • 6. Repassar informações para Nível 3 (N3) - CERMOB.

    Compile todas as informações coletadas e os resultados dos testes em um ticket ou relatório para serem repassados ao suporte de Nível 3, Rômulo da CERMOB.

    Link da documentação completa API:

    • Documentação da API.

    Este procedimento detalha os passos necessários para validar a disponibilidade e funcionalidade da API controladas pela CERMOB, garantindo que todas as etapas sejam seguidas de forma organizada e completa.
